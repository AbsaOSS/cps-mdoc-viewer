**Guidelines for project page authors:**

- **Reference Variety:** Not all projects will have the same types of low-level references. Determine what references your users may need to understand and effectively use your project.
- **Dynamic vs Static:** When mentioning tools or APIs, clarify if they are dynamically generated or static. This helps users know whether they need to regenerate docs after updates.
- **Up-to-date Information:** Especially with APIs and configurations, it's paramount to keep documentation updated. Always cross-check after project updates.
- **Hyperlink Effectively:** If there's additional information on other platforms or sites (like external API docs), make sure to provide direct links. Ensure these links are kept current.
- **Clarity and Brevity:** Low-level references are often technical. Ensure clarity and avoid jargon unless absolutely necessary. Use clear subheadings for different references to make navigation easier.
- **Use Examples:** Whenever possible, provide small snippets or examples on how to use an API call, configuration setting, or other reference. It aids comprehension.
